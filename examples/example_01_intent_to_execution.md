# Exemple 01 — De l’intention à l’exécution

## Intention (I)
« Je souhaite publier un dépôt GitHub minimal documentant ma méthode. »

## Clarification (φ → C)
Voir :
- `contrat/GOAL.md`
- `contrat/SPRINT_BACKLOG.md`
- `contrat/DEFINITION_OF_DONE.md`

## Compilation (κ → F)
Les items du backlog sont compilés en phases dans `task_plan.md`
selon une correspondance stricte un-à-un.

## Exécution (ε → E)
L’exécution suit la discipline *planning-with-files* :
- les découvertes sont consignées dans `findings.md`
- les preuves sont consignées dans `progress.md`
- les phases sont marquées comme *complete* une fois
  les critères de la Definition of Done satisfaits

## Preuve de complétion

Cet exemple correspond aux phases 1 à 3 du sprint définies dans `task_plan.md`.

Toutes les phases sont marquées comme *complete* et sont traçables via :
- `findings.md`
- `progress.md`
